# Contributing to AgentCounsel Hub

AgentCounsel Hub is intended to become an open legal research infrastructure project. Contributions should improve access, structure, searchability, source transparency, and AI-readability of public legal information.

## Contribution types

Useful contributions include:

- Case data source adapters
- Citation parsing improvements
- Search/filter improvements
- Case page UI improvements
- Practical guidance source entries
- Metadata corrections
- Licensing/reuse notes
- Accessibility improvements
- AI-readable endpoint improvements
- Documentation
- Tests

## Contribution principles

1. Preserve source attribution.
2. Do not add third-party full text unless reuse is clearly allowed.
3. Clearly mark AI-generated summaries or labels.
4. Do not present AI output as legal advice.
5. Prefer structured metadata over vague summaries.
6. Keep the platform usable without accounts.
7. Keep public legal information accessible without proprietary lock-in.

## Submitting legal data or guidance

When adding a source, include:

- Source name
- Source URL
- Source type
- Jurisdiction, if applicable
- License or terms URL, if known
- Whether full text can be stored
- Whether only metadata/linking is recommended
- Attribution requirements

## AI-generated contributions

AI-assisted contributions are allowed, but they must be labeled and verifiable.

For AI-generated summaries, tags, or treatment signals, include:

- The source material used
- The field generated
- Review status
- Any confidence or uncertainty note

Do not submit AI-generated legal analysis as authoritative.

## Treatment signals

Citator-like treatment signals require special care.

A proposed treatment signal should include:

- Citing case
- Cited case
- Proposed treatment label
- Evidence passage or source pointer
- Confidence level
- Whether a human reviewer checked it

Experimental labels should remain visibly experimental until reviewed.

## Development workflow

Suggested workflow:

1. Open an issue or comment on an existing issue.
2. Create a branch with a clear name.
3. Make focused changes.
4. Add or update docs/tests where needed.
5. Open a pull request with a concise explanation.

## Disclaimers

This project provides legal research infrastructure and public legal information. It does not provide legal advice and does not create an attorney-client relationship.
